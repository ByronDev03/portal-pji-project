### Product Table
Catálogo de planes disponibles que el usuario puede contratar dentro de la plataforma.

| Campo            | Tipo                        | Descripción |
|------------------|-----------------------------|------------|
| product_id       | CHAR(36) NOT NULL (PK)      | Identificador único del producto generado como UUID. Permite identificar cada plan de forma independiente. |
| name             | VARCHAR(150) NOT NULL       | Nombre del plan mostrado al usuario en la interfaz. |
| description      | VARCHAR(255) NOT NULL       | Descripción del plan y sus beneficios. |
| min_monthly_rent | DECIMAL(10,3) NOT NULL      | Valor mínimo de renta permitido para aplicar al plan. |
| max_monthly_rent | DECIMAL(10,3) NOT NULL      | Valor máximo de renta permitido para el plan. |
| active           | TINYINT(1) DEFAULT 1        | Indica si el plan está disponible. Permite desactivarlo sin eliminarlo. |
| created_at       | DATETIME DEFAULT CURRENT_TIMESTAMP | Fecha de creación del producto. |
| updated_at       | DATETIME DEFAULT CURRENT_TIMESTAMP | Fecha de última actualización del producto. |

### Session Table
Registra las sesiones de usuario para auditoría, trazabilidad y seguridad.

| Campo       | Tipo                        | Descripción |
|------------|-----------------------------|------------|
| session_id | CHAR(36) NOT NULL (PK)      | Identificador único de la sesión (UUID). |
| customer_id| CHAR(36) NOT NULL (FK)      | Referencia al cliente que inició la sesión. |
| ip_address | VARCHAR(45) NOT NULL        | Dirección IP del usuario (compatible con IPv4 e IPv6). |
| user_agent | VARCHAR(255) NOT NULL       | Información del navegador/dispositivo del usuario. |
| status     | VARCHAR(20) DEFAULT 'active'| Estado de la sesión (active, expired, closed). |
| started_at | DATETIME DEFAULT CURRENT_TIMESTAMP | Momento en que inicia la sesión. |
| ended_at   | DATETIME DEFAULT CURRENT_TIMESTAMP | Momento en que finaliza la sesión. |
| created_at | DATETIME DEFAULT CURRENT_TIMESTAMP | Fecha de creación del registro. |
| updated_at | DATETIME DEFAULT CURRENT_TIMESTAMP | Fecha de última actualización. |

### Payment Table
Registra las transacciones realizadas por los clientes al contratar un producto.

| Campo       | Tipo                        | Descripción |
|------------|-----------------------------|------------|
| payment_id | CHAR(36) NOT NULL (PK)      | Identificador único del pago (UUID). |
| customer_id| CHAR(36) NOT NULL (FK)      | Cliente que realiza el pago. |
| product_id | CHAR(36) NOT NULL (FK)      | Producto contratado. |
| amount     | DECIMAL(10,3) NOT NULL      | Monto pagado. |
| currency   | CHAR(3) DEFAULT 'MXN'       | Moneda del pago. |
| method     | VARCHAR(30) NOT NULL        | Método de pago (card, transfer, etc.). |
| status     | VARCHAR(20) DEFAULT 'pending' | Estado del pago (pending, paid, failed). |
| external_ref| VARCHAR(100) NOT NULL      | Referencia externa (ej: Stripe ID). |
| paid_at    | DATETIME DEFAULT CURRENT_TIMESTAMP | Fecha en que se completó el pago. |
| created_at | DATETIME DEFAULT CURRENT_TIMESTAMP | Fecha de creación. |
| updated_at | DATETIME DEFAULT CURRENT_TIMESTAMP | Fecha de actualización. |

### Verification Table
Gestiona los procesos de verificación del usuario y validación de pagos.

| Campo        | Tipo                        | Descripción |
|-------------|-----------------------------|------------|
| verification_id | CHAR(36) NOT NULL (PK)  | Identificador único del proceso de verificación. |
| customer_id | CHAR(36) NOT NULL (FK)      | Cliente asociado al proceso. |
| session_id  | CHAR(36) NOT NULL (FK)      | Sesión en la que ocurre la verificación. |
| payment_id  | CHAR(36) NOT NULL (FK)      | Pago asociado a validar. |
| type        | VARCHAR(50) NOT NULL        | Tipo de verificación (OTP, KYC, biometría). |
| status      | VARCHAR(50) NOT NULL        | Estado del proceso (pending, approved, rejected, expired). |
| attempts    | INT DEFAULT 0               | Número de intentos realizados. |
| expires_at  | DATETIME DEFAULT CURRENT_TIMESTAMP | Fecha de expiración del proceso. |
| verified_at | DATETIME DEFAULT CURRENT_TIMESTAMP | Fecha en que se completó la verificación. |
| created_at  | DATETIME DEFAULT CURRENT_TIMESTAMP | Fecha de creación. |
| updated_at  | DATETIME DEFAULT CURRENT_TIMESTAMP | Fecha de actualización. |
